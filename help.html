Purpose

The document describes the source code analyzer. It describes how to setup and use this application. This analyzer in its current state is not complete and has number of unsupported features. This analyzer only works on Javascript and HTML code.

Design

Analyzing a HTML File:

- First the analyzer scans the current working directory to determine all the files in it. From the list of files it extracts the HTML files and scans the HTML file for all the resources in that project file. This way we populate the project resources.

Analyzing a Javascript File:

- Once we have the project resources, we scan all the javascript files and find all the classes and methods in that class.

- We also scan all the files and determine the global variables in the project

- Once we have a list of all the classes we determine the class hierarchy, composition. This list helps us populate the views tab.

- Using the Znode we display composition, class hierarchy and file content.


Implementation:

Workspace:

There is a large workspace area where all the information is displayed. This workspace has not been separated for different views. But can be done as part of the future enhancement. A horizontal and vertical scroll bars are provided to scroll through the workspace.

Main Menu:

The Main menu has four main options:

1. Open

2. New

3. Save

4. Help

Open:

When a user selects Open option . The user can either scan a current project or view a saved Project. By clicking on the saved project the views that were saved from a previous scan procedure will be displayed.

When the user selects the Scan option then user must enter the working directory in the main menu. Once you enter the path and hit enter. The user is notified if the working directory has been accepted or not.

Scan option:

After selecting the Open option from the main menu and clicking on the “scan” button. The source code analyzer will find all the resources in the project and display the following views.

1. Project resources

2. Classes and Methods

3. Class hierarchy

4. Composition

5. Variables

Project Resources:

This lists all the files that were found by scanning the contents of the HTML files in the directory. The Title of the project is the title of the main webpage. As a future enhancement we should be able to determine the main HTML file name. By clicking on – and + image next to the project name we can hide or show the project resources. Also clicking on the file name will take you to contents of the file . We can view multiple source files at the same time. The Contents of the file are read using the php commands for file manipulation.

Viewing the source file:

Clicking on the file will display the contents of the file. The title bar includes the name of the file. The user can also click on “V “ and save the changes made to the file, hide the file or close the file from the workspace.

The user can view multiple source files at any given time. No search option is provided since browser has a powerful search option builtin. As a future enhancement we should implement a way to display a line number. A column next to the left of the source code has been added but was not completed in time.

Views

The Remaining views are:

  • Class and Method view

  • Class heirarchy view
  • Composition view
  • Variables views

Class/Method View:

By clicking on Class and Method View , All classes in the project are displayed. Clicking on the one of the classes will display All the methods in the class. What line number the methods span. It also draws a tree of from where that method with the line number and code. This view is only available when the user clicks on it and is not removed from the workspace. The user can go back to this view by clicking on the class at any time.

Class Hierarchy

This is displayed on the left most portion of the workspace. In order to display this tree a complex algorithm was implemented which displays all the nodes on the tree in proper sequence. It displays all the method in each class in the tree. The title of the node is the Class name and Top class is super class and bottom node is the subclass. Clicking on the Class hierarchy tab in the views will bring you this view.

Composition

Clicking on the Composition tab displays all the classes that are instantiated in another class. Click on the class name will display the class names from which this class is called and the line number of the code and the name of the variable assigned. In case instantiated without a variable name then only the line number will appear. This view is only generated when the user clicks on the class and come back to this view any time by just clicking on the class.

Variables view

This displays the list of all the global variables in the project. Clicking on one of the global variables will display the file, line number and the code from which it is called. If there multiple references of the variable from the same file then it will show in the same node.

Save Option

This option in the main menu lets the user save the work space information. This can let the user take a snapshot of the project as it development progresses or changes. Opening a saved project doesn’t display the source code files.

When the user clicks on Save option, a “Save Project as” window appears and by entering the name of file with a .json extension will save the entire workspace.

New Option:

Clears the workspace and Project resources and lets the user create a new project manually or open another project.

Help Options:

Shows you this document.

Setup

- First we need to install xampp on the machine.

- Download the latest version of the code from github. The code is available at https://github.com/prshukla/cyse_final_project

- Copy this folder into the htdocs directory

- Make sure there is source code you want to analyze is located in a directory where the webserver has access to it.
